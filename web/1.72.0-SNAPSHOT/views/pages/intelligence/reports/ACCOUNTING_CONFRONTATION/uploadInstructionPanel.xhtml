O confronto contábil permite ao cliente realizar a validação de notas a partir de um arquivo texto. Para utilizar, gere um arquivo conforme o leiaute abaixo, compacte em formato ZIP e faça o upload nessa tela. No final do processamento será disponibilizado um arquivo no mesmo leiaute com o resultado dessa validação.

O arquivo deve conter as informações abaixo, separadas por pipeline.
  • 01: Fixo C100
  • 02: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 03: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 04: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 05: Modelo da nota. NeoGrid vai considerar somente o modelo 55 - NF-e.
  • 06: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 07: Série do documento fiscal
  • 08: Número do documento fiscal
  • 09: Chave da Nota Fiscal Eletrônica
  • 10: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 11: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 12: Valor total do documento fiscal
  • 13: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 14: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 15: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 16: Valor total das mercadorias e serviços
  • 17: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 18: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 19: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 20: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 21: Valor da base de cálculo do ICMS
  • 22: Valor do ICMS
  • 23: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 24: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 25: Valor total do IPI
  • 26: Valor total do PIS
  • 27: Valor total do COFINS
  • 28: Em branco. Caso tenha valor o sistema vai desconsiderar.
  • 29: Em branco. Caso tenha valor o sistema vai desconsiderar.
